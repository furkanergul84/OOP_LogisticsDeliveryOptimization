## 📌 Project Description

This project is a **Logistics and Delivery Optimization System** developed as a take-home assignment for the Object Oriented Programming course.  
The aim of the project is to design and implement a logistics simulation by applying **Object-Oriented Programming (OOP)** principles and progressively enhancing the system through multiple development stages.

The project is implemented in **three stages**, each building upon the previous one, in accordance with the assignment requirements.

---

## 🧱 Project Stages

### 🔹 Stage 1 – Design (`S1_Design`)
This stage focuses on the architectural design of the system.

- Abstract base class (`LogisticsObject`)
- Core entities such as vehicles and logistics issues
- Usage of inheritance, abstraction, composition, and aggregation
- UML/Class diagram describing system structure

This stage defines a clean and extensible object-oriented design.

---

### 🔹 Stage 2 – Basic Implementation (`S2_BasicImplementation`)
This stage implements a working logistics simulation based on the Stage 1 design.

- Turn-based simulation engine
- Interaction between vehicles and logistics issues
- Encapsulation of entity states (status, active)
- Conflict detection and resolution logic
- Factory-style object creation for predefined issue types

This stage demonstrates a functional and interactive system.

---

### 🔹 Stage 3 – Advanced Algorithms (`S3_AdvancedAlgorithms`)
This stage enhances the system with advanced algorithmic features.

- Algorithmic decision-making using strategy selection
- Priority-based issue resolution
- Performance scoring and optimization
- Random (stochastic) operational events
- Dynamic behavior based on system state

This stage transforms the system into an intelligent logistics optimization simulation.

---

## 🧠 Object-Oriented Principles Used

- **Abstraction:** Common behaviors defined via abstract base classes
- **Inheritance:** Shared functionality across related classes
- **Encapsulation:** Internal states managed through properties
- **Polymorphism:** Dynamic strategy selection during runtime
- **Composition:** Capabilities used to enhance vehicle behavior
